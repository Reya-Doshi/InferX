# Security Policy - InferX

We take the security of InferX and its deployments seriously. This document outlines our vulnerability reporting policy and disclosure process.

---

## Supported Versions

We actively maintain security patches for the following versions:

| Version | Supported |
| --- | --- |
| v1.0.x | Yes (Current Stable) |
| < v1.0.0 | No |

---

## Reporting a Vulnerability

**Please do not report security vulnerabilities via public GitHub issues.**

If you discover a security vulnerability in InferX, please report it by emailing us at `security@inferx.ai`.

In your report, please include:
1.  A description of the vulnerability and its potential impact.
2.  Detailed steps to reproduce the issue (proof-of-concept script).
3.  Any potential mitigations or workarounds.

---

## Our Response Process

1.  **Acknowledgment:** We will acknowledge receipt of your report within 24 hours.
2.  **Investigation:** We will investigate the issue and communicate findings with you within 3 business days.
3.  **Patching:** We aim to release a patched version and coordinate a public advisory within 14 days of confirmation.
4.  **Advisory:** We publish security advisories on our GitHub security page.
